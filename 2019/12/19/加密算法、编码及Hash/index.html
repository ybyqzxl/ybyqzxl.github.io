加密

对称加密

原理：使用密钥和加密算法对数据进行转换，得到的数据为密文；使用密钥和解密算法对密文进行逆向转换，得到原数据。

经典算法：

  • DES（密钥短，基本弃用）
  • AES（密钥长，主流）

非对称加密

原理：使用公钥对数据进行加密得到密文；使用私钥对数据进行解密得到原始数据。

（加密算法通过加密密钥【公钥】进行加密，通过加密算法使用解密密钥【私钥】得到原数据）

场景：

A 通过 B 加密密钥加密，发给 B ，B 通过解密密钥进行解密。

私钥可以推算出公钥，因此私钥需要自己保存。

经典算法：

  • RSA （用于加解密）

  • DSA （专门用作签名，具有速度优势）

    用途：签名与验证

使用私钥进行签名，然后使用公钥进行验证。

加密并且签名验证

编码

将数据换一种格式进行表示，同时可以转换回来。

Base64（编码，非加密）

  • 将二进制数据转换为 64 个字符组成的字符串的编码算法（6位一个字符）
    • 二进制数据（非文本数据）

用途：

  • 让数据具有字符串所具有的特性，比如可以放在 URL 中传输、可以保存到文本文件。
  • Base64 加密传输图片并不安全及高效，因为 Base64 并不是加密，同时将数据变大因此也不高效。

URL Encoding（Base64 变种）

  • 将 URL 中的保留字符使用百分号 % 进行编码
  • 目的：消除歧义，避免解析错误

压缩与解压缩

  • 压缩：将数据换一种方式进行存储，减小存储空间
  • 解压缩：将压缩后的数据还原成未解压时的数据。
  • 常见压缩算法：DEFLATE、JPEG、MP3

序列化

  • 序列化：把数据对象（一般为内存中的对象）转换为字节序列的过程。
  • 返序列化：把字节序列重新转换为内存中的对象。
  • 目的：让内存中的对象可以被存储和传输

Hash

  • 定义：把任意数据转换成指定大小范围的数据。
  • 作用：摘要、数字指纹。
  • 经典算法：MD5、SH1、SHA256 等。

用途：

  • 数据完整性验证
  • 快速查找：hashCode() 和 HashMap
  • 隐私保护
